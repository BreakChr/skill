# mihaly-csikszentmihalyi-skill

> *"The best moments in our lives are not the passive, receptive, relaxing times… The best moments usually occur if a person's body or mind is stretched to its limits in a voluntary effort to accomplish something difficult and worthwhile."*

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

**米哈里·契克森米哈赖的最优体验思维框架。用心流（Flow）的底层逻辑拆解专注状态、创造力激发和幸福感来源。**

基于《心流：最优体验心理学》《发现心流》《创造力》等著作及50年实证研究，提炼为可运行的思维框架。

## 使用示例

### 问：我总是分心，无法专注工作

回答会先用"精神熵诊断"帮你识别注意力碎片化的来源——手机通知、多标签页、焦虑念头——然后引导你设计一个"低精神熵环境"：关闭通知、单任务、设置清晰目标。再用"心流通道"检查当前任务是不是挑战太低或太高，帮你调整到最佳匹配。

### 问：为什么我周末休息了反而更累？

回答会用"享乐 vs 乐趣"模型拆解：被动消费（刷剧、刷短视频、无目的浏览）看似在休息，实际上是你把注意力交给了外部算法控制，体验质量很低。然后给出"主动休闲"的具体建议——一项需要你适度投入技能的活动，如运动、乐器、创作、深度阅读。

### 问：工作太无聊了怎么办？

回答会用"心流设计五步法"重构工作：先把大任务分解为小目标，每个目标设置即时反馈机制（打了勾、画了进度、计时完成），然后检查挑战与技能的差距——如果技能远高于挑战，试试主动增加难度（学新方法、设定更高标准、帮同事解决更难的问题）。

更多场景在 [cases/scenarios.md](cases/scenarios.md)。

## 设计思路

和市面上多数"专注力建议合集"不同，这套框架的出发点是：

1. **从底层机制出发**：不教你怎么用番茄钟，教你"心流产生的前提条件是什么"。你拿到这套判断标准，自己就能设计任何任务的心流体验。

2. **可操作的诊断工具**：每个思维模型附带一个具体的"用法"说明——读完就知道怎么用。

3. **诚实面对局限**：公开标注模型的适用边界和争议点，避免盲目套用。

4. **轻量**：不依赖任何外部文件或API，拷走SKILL.md就能用。
